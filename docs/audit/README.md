# Registros de auditoria

Este diretório contém os registros da auditoria read-only realizada sobre o scout-fabric no estado `4e3ca13` (merge da PR #19), produzidos em uma sessão única que percorreu o repositório do mapeamento de drift até decisões arquiteturais consolidadas.

## Frase canônica

**Estes são registros de decisão de auditoria, não documentação canônica.** As fontes canônicas são `docs/architecture/context.md` e os documentos sob `docs/conventions/`. Onde um registro de auditoria divergir do canônico, o canônico vence — o padrão é o mesmo que `package-creator.md:28-29` aplica ao agente lendo `package.md`: o doc canônico tem a última palavra.

A função destes arquivos é preservar o **fundamento** das decisões — por que cada escolha foi feita, contra quais alternativas, com qual ancoragem em fato. Quem aterrissar uma decisão no doc canônico (ex.: cravar o invariante "exclusivo-da-fábrica" no `context.md`, ou escrever o `§8.b` em `package.md`) usa estes registros como **insumo**, não como autoridade. A regra de execução é: re-verificar cada citação ancorada antes de transcrever para o canônico — o grau de escrutínio sobe com o grau de permanência.

## Origem dos arquivos regenerados

Os artefatos `01-drift-report.md` e `03-cobertura-teste.md` foram **regenerados da transcrição da sessão original** porque viveram apenas como respostas inline no chat — nunca tocaram `/tmp` na sessão de auditoria. Quando esta sessão de execução começou, o `/tmp/scout-fabric-audit/` continha apenas 3 dos 5 artefatos esperados; os dois ausentes foram reconstruídos com a regra "conteúdo vem da transcrição, mas cada citação é re-ancorada contra o disco atual — onde a transcrição e o disco divergirem, o disco vence". O resultado: ambos têm cites mais precisos que os originais (quatro target-swaps e cites errados foram corrigidos inline durante a regeneração). Os outros três (`02-tres-formas.md`, `04-decisoes-pendentes.md`, `05-decisoes-tomadas-adr.md`) foram movidos de `/tmp` sem regeneração — já existiam como arquivos persistidos.

## Como ler estes arquivos

A ordem narrativa é cronológica (`01` → `05`) e cumulativa: cada arquivo pressupõe o anterior. O **`05-decisoes-tomadas-adr.md` é o destino prático** — quem precisar agir a partir desta auditoria começa por ele e usa os outros como referência ancorada. O `04-decisoes-pendentes.md` é o caminho intermediário entre o mapa (01–03) e o registro (05); contém o raciocínio que levou às decisões e é relevante quando uma decisão precisa ser reaberta (o que exige entender a opção rejeitada, não só a escolhida).
