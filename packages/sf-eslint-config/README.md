# @fabio.caffarello/sf-eslint-config

Shared ESLint flat config used by `@fabio.caffarello/sf-*` packages and by
projects generated by the scout-fabric.

## What it ships

A single array (default export) of [ESLint flat config](https://eslint.org/docs/latest/use/configure/configuration-files-new)
entries, in this order:

1. Common `ignores` (build, deps, cache, coverage, tmp).
2. `@nx/eslint-plugin` flat presets (`flat/base`, `flat/typescript`,
   `flat/javascript`) — gives downstream access to
   `@nx/enforce-module-boundaries` and Nx-aware lint.
3. Workspace-wide opinionated rules:
   - `eqeqeq: ['error', 'always']`
   - `no-console: ['warn', { allow: ['warn', 'error'] }]`
4. [`eslint-config-prettier`](https://github.com/prettier/eslint-config-prettier)
   last, to disable rules that would conflict with Prettier.

## Usage

```js
// eslint.config.mjs of a downstream project
import sf from '@fabio.caffarello/sf-eslint-config';

export default [
  ...sf,
  // local overrides go here (e.g. depConstraints for
  // @nx/enforce-module-boundaries, typed-lint with projectService)
];
```

## What it does NOT include

- `@nx/enforce-module-boundaries` rule activation — that needs
  `depConstraints` specific to each workspace. Configure it in the
  consumer's flat config after spreading this one.
- Typed-aware rules (`@typescript-eslint/no-floating-promises` etc.) —
  those need `parserOptions.projectService` + `tsconfigRootDir`,
  which are workspace-specific.
